# Gate Template

Define each gate with:

- Gate name.
- Behavior to enforce.
- Surface: checklist, script, test, hook, CI job, final-response check, or
  policy file.
- Trigger condition.
- Pass signal.
- Fail signal.
- False-positive risk.
- Bypass or emergency override policy.
- Owner and maintenance location.

For v1 of this suite, templates are inert until a separate implementation task
installs them.
