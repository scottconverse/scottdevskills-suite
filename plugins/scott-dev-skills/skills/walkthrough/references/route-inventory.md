# Route Inventory

Use this when a walkthrough needs coverage discipline.

Capture:

- Static route definitions and dynamic route patterns.
- Navigation links, tabs, drawers, menus, breadcrumbs, and deep links.
- Authenticated, unauthenticated, permissioned, empty, loading, error, and
  success states.
- Primary user workflows and secondary settings/admin flows.
- Backend/API routes each screen depends on.

Mark each route or workflow as explored, partially explored, blocked, or not in
scope. If repeated screens share a component pattern, audit one representative
example and name the pattern scope.
