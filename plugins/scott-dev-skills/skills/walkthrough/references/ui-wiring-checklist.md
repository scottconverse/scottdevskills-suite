# UI Wiring Checklist

Check for:

- Buttons, links, icons, and menu items that do nothing.
- Clickable-looking elements that are inert.
- Forms without validation, persistence, or error handling.
- UI that appears real but uses hardcoded, mocked, or stale data.
- Frontend actions that do not reach expected API/service/storage paths.
- Backend capabilities present in code but missing from UI.
- Broken navigation, route guards, refresh behavior, and back/forward behavior.
- Missing loading, empty, disabled, success, and failure states.
- Console errors, hydration mismatches, failed requests, and broken assets.
- Desktop/mobile overlap, clipping, unreadable text, and inaccessible controls.
