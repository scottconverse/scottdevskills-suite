Built for the work between idea and ship.

ScottDevSkills is not magic and it does not pretend to replace judgment. It gives Codex sharper operating procedures for the moments where vague help is not enough: reviewing a risky fix, proving a UI is wired, keeping a long run on track, or making prompts testable.

Audit Lite

Fast, scoped review for one fix or a small diff. Findings first, tied to evidence.

Audit Full

Deep release/readiness audit across code, UX, docs, tests, and QA behavior.

Walkthrough

Browser-driven UI wiring review for routes, buttons, forms, console errors, and real behavior.

Agent Pipeline

Manifest-driven project execution with intake, validation, run status, and stop conditions.

Prompt Quality

Prompt linting, regression case design, eval planning, and release checks for prompt changes.

Context Discipline

Long-session hygiene, large-output control, handoffs, and careful pre-edit analysis.

What it is good at Turning Codex into a more disciplined development partner when there is enough risk to justify process: audits, release checks, UI truth, structured runs, prompt regressions, and handoffs.
What it is not It is not a guarantee that every bug will be found. It is not a replacement for tests, production monitoring, or human ownership. It is a better operating loop for getting evidence before confidence.

How the skills work together.

1. Shape Intake, Context Discipline, and Prompt Quality turn unclear work into testable intent.
2. Structure Pipeline Init, New Run, and Validate Manifest create controlled execution boundaries.
3. Execute Run Pipeline moves staged work forward while preserving gates and evidence.
4. Inspect Audit Lite, Walkthrough, and Audit Full check the work at the right depth.
5. Harden Hardgate Templates turn repeated misses into explicit checks.

Technical model.

Plugin package Marketplace metadata points Codex at the installable scott-dev-skills package.
Skill entrypoints Each SKILL.md defines trigger boundaries, workflow, output shape, and failure handling.
Progressive references Detailed rubrics, checklists, report contracts, and templates stay outside the initial skill body until needed.
Pipeline payloads Pipeline skills include manifests, roles, policy scripts, and templates for structured project runs.
Validation The suite ships a validator plus trigger regression cases to keep names, references, and skill structure honest.
Default safety Audit and walkthrough skills review by default. Gate templates are inert unless a later task installs enforcement.
The pitch is simple: less "looks fine," more "here is what I checked, here is what failed, and here is the next concrete move."

CLI

A first-time CLI install is two short steps. If the marketplace is already configured, refresh it before installing so Codex uses the current snapshot. If the plugin is already installed, remove it before refreshing and reinstalling. 

codex plugin marketplace add scottconverse/scottdevskills-suite --ref main --sparse .agents/plugins/marketplace.json --sparse plugins/scott-dev-skills
codex plugin add scott-dev-skills@scottdevskills
codex plugin marketplace upgrade scottdevskills
codex plugin add scott-dev-skills@scottdevskills
codex plugin remove scott-dev-skills@scottdevskills
codex plugin marketplace upgrade scottdevskills
codex plugin add scott-dev-skills@scottdevskills
