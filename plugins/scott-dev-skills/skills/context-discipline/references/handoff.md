# Handoff

A useful handoff contains:

- Goal and current status.
- Files changed or inspected.
- Commands/checks run and their outcomes.
- Open blockers and decisions.
- Next safe action.
- Important constraints from user, repo instructions, or policy.
- Pointers to durable artifacts, logs, screenshots, reports, or run directories.

Keep handoffs factual. Do not include speculation unless clearly labeled.
