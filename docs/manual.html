Overview

ScottDevSkills is a Codex plugin for evidence-first development work. It gives Codex focused operating procedures for audits, UI walkthroughs, manifest-driven pipeline runs, prompt quality checks, context discipline, and optional gate design.

The suite is built around one principle: confidence should come from evidence, not polished narration. A good workflow should leave behind what was inspected, what was verified, what failed, what is still unknown, and what should happen next.

Best use Higher-risk development work that needs a repeatable review, execution, or handoff loop.
Default stance Gather evidence first. Repair only when repair is explicitly requested.
Design Narrow skill entrypoints with heavier references loaded only when needed.

Installation

ScottDevSkills is distributed as a Codex marketplace plugin. Install the marketplace first, then install the plugin named scott-dev-skills.

Codex app install

Add or refresh marketplace repository scottconverse/scottdevskills-suite, install scott-dev-skills, then start a fresh Codex thread.

CLI install

First-time CLI setup uses two steps. Once the marketplace is configured, refresh it before plugin install so Codex uses the current snapshot. 

codex plugin marketplace add scottconverse/scottdevskills-suite --ref main --sparse .agents/plugins/marketplace.json --sparse plugins/scott-dev-skills
codex plugin add scott-dev-skills@scottdevskills
codex plugin marketplace upgrade scottdevskills
codex plugin add scott-dev-skills@scottdevskills

Troubleshooting

  • Start a fresh Codex thread if newly installed skills do not appear.
  • Use the plugin id scott-dev-skills@scottdevskills when reinstalling or removing it.
  • If the marketplace is already configured, refresh scottdevskills before installing from it.

How The Skills Work Together

The suite covers the lifecycle from unclear intent to structured execution, inspection, and hardening. You do not need every skill for every task; the point is to pick the smallest workflow that matches the risk.

1. Shape Intake, Context Discipline, and Prompt Quality turn unclear work into testable intent.
2. Structure Pipeline Init, New Run, and Validate Manifest create controlled execution boundaries.
3. Execute Run Pipeline moves staged work forward while preserving gates and evidence.
4. Inspect Audit Lite, Walkthrough, and Audit Full check work at the right depth.
5. Harden Hardgate Templates turn repeated misses into explicit checks.

Technical Model

Package Structure

The marketplace file exposes the plugin to Codex. The plugin manifest declares metadata, version, capabilities, and the skill location. Each skill folder contains a SKILL.md entrypoint and optional references.

Execution Behavior

Most skills are procedural: they tell Codex what to inspect, how to classify evidence, and what output shape to produce. Pipeline skills may create or update project files when the user asks for setup, scaffolding, or execution. Audit and walkthrough skills default to review mode. Hardgate Templates are inert by default.

Evidence Standards

A finding should identify the affected file, route, workflow, prompt, run artifact, or UI element; expected behavior; observed behavior or concrete risk; impact; likely cause when supportable; and a suggested fix or test.

If evidence is incomplete, ScottDevSkills should preserve that uncertainty instead of turning a hunch into a defect.

Skill Reference

Audit Lite Fast review of a bug fix, scoped diff, a few touched files, or a pre-merge sanity check. Output should be brief and findings-led.
Audit Full Broad readiness review across engineering, UI/UX, docs, tests, and QA. Best for release gates and handoffs.
Walkthrough Browser and source inspection for UI wiring, routes, controls, forms, console errors, network failures, and persistence.
Agent Pipeline Router for manifest-driven work: initialize, capture, create, validate, run, resume, inspect, or scaffold.
Pipeline Init Prepares a repo with templates, policy scripts, run directory conventions, and starter guidance.
Intake Captures loose intent as durable pipeline input without starting execution.
New Run Creates a fresh run skeleton and prepares manifest/scope-lock materials.
Validate Manifest Checks schema, missing fields, path issues, scope locks, and policy-shape problems before execution.
Run Pipeline Executes or resumes a manifest-driven run while respecting gates, policy checks, verification, and stop conditions.
Show Run Status Read-only summary of run id, current stage, blockers, decisions, and next valid action.
Audit Init Scaffolds audit protocols, audit gates, and five-lens self-audit materials.
Prompt Quality Reviews prompts for ambiguity, output contracts, tool hazards, injection exposure, and regression/eval coverage.
Context Discipline Manages long sessions, large output, handoffs, and careful pre-edit tracing for non-trivial code work.
Hardgate Templates Designs policy checks, preflight scripts, final-response checks, or hook templates without installing enforcement by surprise.

Operating Guidance

  • Start with the narrowest skill that matches the task.
  • Do not use audit skills as repair skills unless repair is explicitly requested.
  • Prefer runtime evidence when reviewing UI behavior.
  • Prefer regression cases when reviewing prompts.
  • Prefer manifests and scope locks for work that spans multiple stages.
  • Preserve uncertainty instead of overstating findings.

ScottDevSkills v0.1b is an early beta. The plugin is installable and validated, and its workflows are intentionally conservative. The next improvements should come from real use: better trigger regression cases, sharper output contracts, and examples from successful audits, walkthroughs, and pipeline runs.
